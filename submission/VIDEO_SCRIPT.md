# Verified Engineering Studio — demo video script

Target: 2:35–2:50, English narration, 16:9, 1080p, captions enabled. Record the deployed public
site, not a local development build. Keep the cursor deliberate and avoid dead time.

## 0:00–0:15 — Problem and hook

**Picture:** New product hero with the real Laurons wave field, package-verification badge and
period-mean card. Move directly into the studio with **Explore the CFD case**.

**Narration:**

> Engineering AI can sound certain long before the engineering is certain. Verified Engineering
> Studio changes that: deterministic checks remain in control, while GPT-5.6 explains only the
> evidence it is given.

## 0:15–0:35 — Cross-domain product

**Picture:** Module cards, then enter the studio. Briefly point to CFD and Robotics.

**Narration:**

> VES is a public, read-only evidence workbench. The same small module contract supports very
> different workflows: an OpenFOAM resistance study today, and an Isaac Sim skill-verification
> handoff next. The public app cannot start a solver, simulator or robot.

## 0:35–1:02 — Immutable CFD evidence

**Picture:** CFD case header, package SHA and four key metrics. Open the complete metric ledger,
then use the evidence viewer to enlarge the wave image and play the mesh video in fullscreen.

**Narration:**

> This Laurons II run is no longer a hard-coded demo object. It is loaded from a VES Evidence
> Package with typed schemas, publication rights, exact byte sizes and SHA-256 hashes for the data
> and every project-generated media file. Tampered, undeclared, executable, secret-shaped or
> path-leaking content fails before review.

## 1:02–1:25 — Facts versus methods

**Picture:** Open “Source provenance”, then open “Engineering method basis”. Keep the Versteeg and
ITTC cards readable beside the large media viewer.

**Narration:**

> Case evidence and engineering literature are deliberately separate. The measured force values
> come from the supplied run. Versteeg and Malalasekera’s finite-volume textbook and the ITTC
> procedure justify the verification rules; they never become invented case evidence.

## 1:25–1:58 — Deterministic gates first

**Picture:** Select “Result support”, click “Run verified review”, then show the gate list.

**Narration:**

> The deterministic layer first reproduces force closure, patch balance and the reference
> deviation. It also finds what is missing. Only the fine grid exists, so discretization uncertainty
> remains open, and the sanitized public package is not yet reproduction-complete. Those warnings
> cap the final status at review, regardless of model wording.

## 1:58–2:18 — GPT-5.6 with provenance and cost control

**Picture:** Show the structured summary, evidence-ID chips, next actions and `gpt-5.6 · cache hit`.

**Narration:**

> GPT-5.6 turns the checked bundle into structured findings, caveats and next actions with exact
> evidence IDs. Application code removes unknown citations and enforces the gate status. Identical
> public questions are cached, and hard visitor and daily limits prevent an open demo from becoming
> an open bill.

## 2:18–2:33 — Honest Isaac boundary

**Picture:** Switch to Robotics; show `Handoff pending` and the read-only boundary check.

**Narration:**

> The robotics module is equally explicit: its stable execution export is still pending, so VES
> presents a preview instead of claiming proof it does not yet possess.

## 2:33–2:48 — Codex, architecture and close

**Picture:** Architecture section, then a quick terminal/GitHub cut showing `35 passed`, package
validation and green CI. Include a brief Codex session/repository-history shot.

**Narration:**

> We built the new platform with Codex during Build Week: modular adapters, package integrity,
> deterministic CFD tools, Structured Outputs, tests and deployment. VES does not replace the
> engineer. It makes the evidence behind an engineering decision inspectable.

**End card:** `verifiedengineeringstudio.schares-timo.de` and GitHub repository.

## Recording checklist

- Use the **Result support** prompt first; it is already cached and will not wait on a model call.
- Record at 1440×900 or 1920×1080 with browser zoom 100%.
- Hide bookmarks, notifications, terminal usernames and all environment/secret views.
- Use only repository media and the linked LDPF page; do not copy third-party photographs or SVA
  report pages into the video.
- Add burned-in or YouTube captions and check pronunciation of “Laurons”, “Versteeg”, “GCI” and
  “Isaac”.
- Keep final duration below three minutes and verify audio on headphones and phone speakers.
- Upload publicly or unlisted to YouTube, then paste the final URL into Devpost and test the embed in
  a logged-out browser.
